The Problem

Dark Vessels

Thousands of vessels in the Arabian Sea, Bay of Bengal and wider Indian Ocean run "dark," switching off AIS beacons to hide illegal fishing, piracy, smuggling or grey-zone naval moves.

Limited Visibility

Regional agencies rely on expensive paywalled tools or ad-hoc intelligence and lack access to a public, near-real-time maritime surveillance feed.

Our Solution

Multi-Source Intelligence

Ingests free AIS broadcasts, public Sentinel-2 imagery, and task-on-demand SAR snapshots using advanced fusion algorithms.

AI-Powered Detection

Lightweight ML pipeline flags every satellite-detected ship without matching AIS position (≤2nm/≤30min) with high accuracy.

Open Access

Live web dashboard, REST API, CSV downloads, and weekly email briefs. All code and data released under MIT/CC-BY-SA 4.0.

Technology Stack

Ingest

Python scrapers, terrestrial AIS (AISHub), ESA Sentinel-hub API, Iceye/Umbra SAR quota

Storage

PostgreSQL + PostGIS for geospatial data, S3 object store for imagery and archives

ML Pipeline

PyTorch framework, YOLOv8 fine-tuning, unsupervised track-anomaly detection module

Backend

FastAPI microservices, Docker containers, AWS Lambda cron jobs for automation

Frontend

Next.js/React or Streamlit MVP, MapLibre GL for interactive tile rendering

CI/CD

GitHub Actions pipeline, pytest with ≥80% code coverage requirements

12-Month Roadmap

Months 0-2: Foundation

Cloud infrastructure deployment, AIS scraper operational, basic mapping interface for Arabian Sea coverage

Months 3-4: AI Training

200 Sentinel-2 chips labeled for training, YOLOv8 ship detector achieving mAP ≥ 0.6 performance

Months 5-6: Beta Launch

Correlator flagging daily dark vessels, public beta dashboard with user feedback integration

Months 7-9: Expansion

Coverage extended to Bay of Bengal + IOR corridors, REST API and email briefing system launched

Months 10-12: Scale

1TB AIS stored, ~1,500 detections & ~250 dark alerts daily, 100+ subscribers, Indian Navy pilot demonstration

Open-Source Philosophy

Transparency First

Complete codebase available under MIT license. All derived data shared under CC-BY-SA 4.0. No black boxes, no vendor lock-in.

Community Driven

Open data invites researchers, NGOs, and startups to build add-ons. Collaborative development with maritime security community.

Replicable Framework

Codebase and pipeline designed to be cloned for any littoral region at low cost. Democratizing maritime domain awareness globally.

Security Through Openness

Open-source approach enables community auditing, rapid vulnerability patches, and builds trust with maritime stakeholders.
